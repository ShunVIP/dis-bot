## Архитектура проекта (фиксированная, опираться на неё)

- **Cogs:** в `fun_slesh/`. Каждый модуль экспортирует `async def setup(bot):` и использует slash‑команды (`app_commands`).
- **События:** `utils/events_bus.py` — функции `emit`, `subscribe` для кросс‑модульной логики (ачивки/экономика/алёрты).
- **Планировщик:** APScheduler (таймзона МСК; хранение дат — UTC).
- **БД:** SQLite в `datebase/` (`social.db`, `reminders.db`, `birthdays.db`). Новые таблицы добавляй в существующие БД. Всегда: `CREATE TABLE IF NOT EXISTS`, индексы, миграционно‑безопасно.
- **Логирование:** `utils/logger.py` (loguru → `bot.log`).
- **UX‑паттерн:** долгие операции — `await interaction.response.defer()` + последующие `followup.send()`. Уважай `Unknown interaction` и делай безопасные ответы (см. паттерны в проекте).
- **Таймзоны:** хранить в UTC, показывать пользователю МСК при необходимости.
- **Секреты/конфиг:** через `KGTD.env` (например, `RIOT_API_KEY`). При отсутствии ключа — **офлайн‑режим** с self‑report/локальными данными и явным сообщением об этом.

**Примеры и паттерны внутри проекта (обязательно изучить перед реализацией):**
- `fun_slesh/test_hello.py` — кнопки/опросы/безопасные ответы.
- `fun_slesh/message_and_voice_stats.py` — индексация истории, агрегаты, defer/followup‑хелперы, работа с SQLite.
- `fun_slesh/tools.py`, `fun_slesh/mem_roles.py`, `fun_slesh/rep_and_mood.py`, `fun_slesh/achievements_engine.py` — структура cogs, APScheduler, роли, ачивки.
- `utils/events_bus.py`, `utils/logger.py` — шина событий и логирование.

## Ключевые принципы (всегда соблюдать)

1. **Только бесплатные API**. Любая внешняя интеграция — опциональна, с проверкой наличия ключа и **полным офлайн‑фоллбеком**.
2. **Ни один модуль не правит `main_file.py`**. Всё — через отдельные cogs.
3. **SQLite‑схемы — автосоздание**. Никаких ручных миграций на старте.
4. **Троттлинг и ретраи** для внешних вызовов (обрабатывай `429`/`5xx`, уважаем rate‑limit заголовки).
5. **Стабильность > функционал**: понятные ошибки пользователю, без утечки секретов.
6. **Событийность:** на ключевых действиях отправляй `emit(...)` (для ачивок/экономики/логики).
7. **Код‑стайл:** маленькие функции, понятные имена, docstring, комментарии «что и почему», проверка типов (где уместно). Тестируемость команд — через сценарии проверки в конце ответа.

## Реализуемые модули (минимальные спецификации)

### 1) LoL интеграция — `fun_slesh/lol_stats.py`
- ENV: `RIOT_API_KEY` (если нет ключа — офлайн: self‑report и локальные агрегаты).
- Таблицы (в `social.db`):
  - `lol_accounts(puuid TEXT PRIMARY KEY, summoner TEXT, region TEXT, linked_user_id INTEGER, linked_at TEXT)`
  - `lol_matches(match_id TEXT PRIMARY KEY, puuid TEXT, queue INTEGER, ts TEXT, champ TEXT, role TEXT, lane TEXT, k INTEGER, d INTEGER, a INTEGER, win INTEGER, duration INTEGER, items_json TEXT, perks_json TEXT)`
  - Индексы по `(puuid, ts)`.
- Команды:
  - `/lol_аккаунт ник регион` — линковка + получение PUUID; запись в БД.
  - `/lol_матчи [N]` — загрузка последних N матчей (с троттлингом/ретраями).
  - `/lol_сводка` — агрегаты (винрейт по чемпионам/ролям).
  - `/lol_совет_чемпиона [роль]` — **ML** рекомендация (WinPredictor/BuildRecommender).
- События: `emit("lol_advice_used", champion=..., role=..., user_id=...)`.

### 2) Helldivers 2 War — `fun_slesh/helldivers_war.py`
- Источник: публичные/community эндпоинты War/Operations (без auth). Если источник недоступен — офлайн: «данные недоступны».
- Таблицы (в `social.db`):
  - `hd2_war_snapshots(ts TEXT PRIMARY KEY, factions_json TEXT, planets_json TEXT, ops_json TEXT)`
- Команды:
  - `/hd2_фронт` — текущий фронт + дельта с прошлого снапшота.
  - `/hd2_операции` — активные операции дня.
  - `/hd2_прогноз [часы]` — **ML** прогноз изменения фронта (WarForecast).
- Планировщик: снапшот каждые 10–15 минут (МСК), защита от дубликатов.
- События: `emit("hd2_forecast_ready", horizon=...)`.

### 3) ETL и фичи (ML‑витрина)
- `ml/etl_lol.py`: из `lol_matches` → `ml_lol_features(puuid, match_id, feat_json, target_win)`.
- `ml/etl_hd2.py`: из `hd2_war_snapshots` → `ml_hd2_features(ts, feat_json, target_delta_front)`.
- Вызывать по APScheduler ночью; лог метрик в `bot.log`.

### 4) ML‑тренеры и предсказатели
- Модели в `ml/models/*.pkl` (`joblib`).
- **LoL/WinPredictor** (LogReg → RandomForest): `/lol_прогноз_победы [роль|чемп]`.
- **LoL/BuildRecommender** (MVP: винрейт/частоты → затем ранжирование): `/lol_совет_чемпиона [роль]`.
- **HD2/WarForecast** (Ridge/Lasso → GBM): `/hd2_прогноз [часы]`.
- Общий админ‑cog `fun_slesh/ml_admin.py`: `/ml_train_all`, `/ml_stats`, `/ml_reload`, `/ml_predict ...` (для отладки).

### 5) Геймплейные фичи (без платных API; привязка к ML позже)
- `fun_slesh/lfg.py`: `/lfg игра режим платформа [через_минут]` (кнопки «вступить/пинг», авто‑истечение, связка с `mem_roles`).
- `fun_slesh/match_reports.py`: `/матч_создать`, `/матч_результат`, `/рейтинг` (подтверждение соперником, анти‑фрод).
- `fun_slesh/challenges.py`: `/челлендж_мне`, `/челлендж_сдать`, `/челлендж_рейты` (событие `challenge_completed`). 
- `fun_slesh/battlepass.py`: `/паспорт`, `/паспорт_миссии`, `/паспорт_награды`.

### 6) Модерация/безопасность (п.6) — позже
- Лёгкая тональность/токс‑классификация (офлайн, sklearn).
- Анти‑спам по аномалиям активности.
- Приватность: без персональных токенов без явного согласия.

### 7) Авто‑отчёты сообщества (п.7)
- Ночные сводки в канал: активность/онлайн, настроение, winrate по играм, тренды Helldivers.
- Текст/эмбеды, без платных внешних сервисов.
