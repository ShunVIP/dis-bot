# 🤖 Модуль пародии — установка и подключение

## Что добавляется

| Файл | Назначение |
|------|-----------|
| `fun_slesh/parody_collector.py` | Сбор сообщений со всех каналов в `datebase/messages.db` |
| `fun_slesh/parody_engine.py` | ML-модели (markovify), генерация фраз, еженедельное дообучение |
| `models/` | Папка с обученными моделями (`<user_id>.json`) — создаётся автоматически |

## Новые команды (6 штук)

| Команда | Кто | Описание |
|---------|-----|----------|
| `/собрать_сообщения` | Админ | Собрать все сообщения со всех каналов |
| `/стиль_статистика [@user]` | Все | Статистика сообщений + топ слов пользователя |
| `/пародия <@user>` | Все | Сгенерировать фразу в стиле пользователя |
| `/мем_фраза <@user>` | Все | Короткая фраза ЗАГЛАВНЫМИ для мема |
| `/дообучить [@user]` | Админ | Обучить/дообучить модели (одного или всех) |
| `/модели_статус` | Админ | Список пользователей и статус моделей |

## Установка зависимости

```bash
pip install markovify
```

Добавь в `requirements.txt`:
```
markovify
```

## Подключение

Файлы `parody_collector.py` и `parody_engine.py` кладёшь в папку `fun_slesh/`.
Бот подхватит их автоматически при следующем запуске (автозагрузка cogs работает).

## Порядок первого запуска

1. Установить зависимость: `pip install markovify`
2. Положить оба файла в `fun_slesh/`
3. Запустить бота
4. В Discord: `/собрать_сообщения` — дождаться завершения (может занять несколько минут)
5. `/дообучить` — обучить модели для всех пользователей с ≥50 сообщений
6. `/пародия @кто-нибудь` — готово!

## Автообновление

Каждое **воскресенье в 03:00 МСК** бот автоматически:
1. Собирает новые сообщения (с даты последнего сбора)
2. Дообучает все модели (объединяет новые данные со старыми: 70% новые / 30% старые)

## Как работает модель

- **markovify** — цепи Маркова второго порядка (`state_size=2`)
- Смотрит на пары слов и учится предсказывать следующее
- Чем больше сообщений — тем точнее стиль речи
- Минимум для нормального результата: **200+ сообщений**
- При дообучении старая модель не удаляется, а объединяется с новой

## Структура БД (messages.db)

```
user_messages       — все отфильтрованные сообщения
collect_checkpoints — чекпоинты по каналам (для инкрементного сбора)
known_users         — user_id → последний известный ник
```

## Заметки

- User ID в Discord не меняется — это основной ключ, ник сохраняется отдельно
- Фильтруются: команды `/`, ссылки, упоминания `@`, сообщения короче 3 слов, боты
- Модели хранятся в `models/<user_id>.json` — можно удалить для переобучения с нуля
