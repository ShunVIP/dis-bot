---

## 🧱 Общая идея

**dis-bot** — многофункциональный Discord‑бот на **Python 3.12** с:

* полной поддержкой **slash‑команд** (`discord.py 2.x`),
* модульной архитектурой (cogs),
* локальным хранением данных в **SQLite**,
* системой событий для ачивок и реактивной логики,
* планировщиками фоновых задач (**APScheduler**).

Бот предназначен для:

* оживления Discord‑сообщества;
* социальной активности;
* геймификации;
* статистики и аналитики;
* утилитарных задач (напоминания, роли, дни рождения).

## 🚀 Запуск

### Требования

* Python **3.12**
* Discord Bot Token

### Установка

```bash
py -3.12 -m pip install -r requirements.txt
```

Создать файл **KGTD.env** рядом с `main_file.py`:

```env
tok=YOUR_DISCORD_BOT_TOKEN
```

Запуск:

```bash
py -3.12 main_file.py
# или
start.bat
```

⚠️ В Discord Developer Portal **обязательно включить**:

* `Message Content Intent`
* `Server Members Intent`

## 🗂 Архитектура проекта

```
dis-bot/
├── main_file.py            # Точка входа, загрузка cogs, sync slash-команд
├── config.py               # Загрузка токена из KGTD.env
├── start.bat               # Быстрый запуск под Windows
├── requirements.txt        # Зависимости

├── fun_slesh/              # Основные slash‑команды (cogs)
├── core/                   # Базовые доменные сервисы (экономика)
├── utils/                  # Инфраструктура (логгер, event bus)
├── scheduled/              # Планировщики задач
├── datebase/               # SQLite базы данных
├── context_menus/          # Context‑menu команды
├── events/                 # Заглушки/ивенты Discord
```

### Ключевые принципы

* 🔌 **Автозагрузка модулей** — все файлы в `fun_slesh/` подключаются автоматически.
* 🔔 **Event Bus** — слабая связанность логики (`emit / subscribe`).
* 💾 **SQLite first** — никакой внешней инфраструктуры.
* 🕒 **MSK‑логика времени** — дэйлики, настроение, напоминания.
