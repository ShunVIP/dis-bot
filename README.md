# ViPik Discord Bot

Discord-бот для одного сервера с уклоном в игры, экономику, статистику, пародии на стиль речи, Steam, напоминания и автоматизацию. Проект рассчитан на схему:

- код пишется на ПК;
- бот постоянно живёт на VPS;
- тяжёлые GPT-модели при необходимости работают с ПК через bridge;
- VPS хранит боевые базы и каждый день добирает новые сообщения.

## Что есть в репозитории

### Точка входа

- `main_file.py` — запуск бота и загрузка всех модулей.
- `config.py` — чтение `KGTD.env`.

### Основные папки

- `fun_slesh/` — slash-команды и listeners.
- `core/` — базовая логика и runtime policy.
- `scheduled/` — фоновые задачи.
- `scripts/` — локальные и серверные служебные скрипты.
- `deploy/systemd/` — шаблоны сервисов для Linux.
- `datebase/` — SQLite-базы.
- `models/` — локальные модели и артефакты.
- `wwm_kb/` — база знаний Where Winds Meet.

## Безопасность и политика данных

### Что не должно попадать в git

В git не должны попадать:

- `KGTD.env`
- любые реальные токены и API-ключи
- `datebase/*.db`
- `models/`
- приватные SSH-ключи
- логи

Это уже прикрыто в [`.gitignore`](D:/dis-bot/.gitignore), но перед каждым `git push` всё равно полезно смотреть `git status`.

### Что уже проверено

В tracked-файлах репозитория сейчас нет:

- реального `KGTD.env`
- приватных ключей
- реальных Discord token / GitHub token / OpenAI key
- SQLite-баз
- `models/`

В репозитории есть только плейсхолдеры в:

- [KGTD.env.example](D:/dis-bot/KGTD.env.example)
- [README.md](D:/dis-bot/README.md)

### Политика работы на VPS

На VPS по умолчанию:

- разрешена обычная работа бота;
- разрешён безопасный ежедневный цикл `сбор сообщений -> Markov`;
- запрещено тяжёлое GPT-обучение;
- запрещена полная тяжёлая `/профилактика`;
- удалённую тяжёлую модель можно подключать с ПК вручную.

Эта логика хранится в [core/runtime_policy.py](D:/dis-bot/core/runtime_policy.py).

## Переменные окружения

Бот читает настройки из файла `KGTD.env` в корне проекта.

Шаблон:

```env
tok=YOUR_DISCORD_BOT_TOKEN
STEAM_API_KEY=YOUR_STEAM_API_KEY
REMOTE_MODEL_API_URL=
REMOTE_MODEL_API_TOKEN=
BOT_SERVER_MODE=true
ALLOW_GPT_TRAINING_ON_SERVER=false
ALLOW_FULL_MAINTENANCE_ON_SERVER=false
ALLOW_REMOTE_MODEL_INFERENCE=true
ENABLE_DAILY_MARKOV_RETRAIN_ON_SERVER=true
ENABLE_DAILY_MARKOV_COLLECTION_ON_SERVER=true
DAILY_MARKOV_RETRAIN_HOUR=3
DAILY_MARKOV_RETRAIN_MINUTE=15
WEB_ADMIN_ENABLED=false
WEB_ADMIN_HOST=127.0.0.1
WEB_ADMIN_PORT=8080
WEB_ADMIN_TOKEN=
WEB_ADMIN_TITLE=ViPik Bot Control
```

### Что значат важные флаги

- `BOT_SERVER_MODE=true` — серверный режим.
- `ALLOW_GPT_TRAINING_ON_SERVER=false` — запрещает GPT-обучение на VPS.
- `ALLOW_FULL_MAINTENANCE_ON_SERVER=false` — запрещает тяжёлую `/профилактика` на VPS.
- `ALLOW_REMOTE_MODEL_INFERENCE=true` — разрешает VPS использовать тяжёлую модель с ПК.
- `ENABLE_DAILY_MARKOV_RETRAIN_ON_SERVER=true` — ежедневное безопасное Markov-дообучение на VPS.
- `ENABLE_DAILY_MARKOV_COLLECTION_ON_SERVER=true` — ежедневный добор новых сообщений перед Markov.
- `DAILY_MARKOV_RETRAIN_HOUR` / `DAILY_MARKOV_RETRAIN_MINUTE` — время по МСК.
- `WEB_ADMIN_ENABLED=true` — включает мини-панель в браузере.

## Что реально работает автоматически

### На VPS

- постоянный запуск бота через `systemd`;

- включение/выключение bridge тяжёлой модели.

## Установка локально с нуля

### Требования

- Windows 10/11
- Python 3.10+
- Git
- OpenSSH client

### Шаги

1. Склонировать репозиторий:

```powershell
git clone https://github.com/ShunVIP/dis-bot.git
cd dis-bot
```

2. Создать `KGTD.env` по образцу [KGTD.env.example](D:/dis-bot/KGTD.env.example).

3. Установить зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

4. Запустить бота:

```powershell
python main_file.py
```

## Тяжёлая GPT-модель с ПК

### Включить

```bat
enable_heavy_models.bat
```

### Выключить

```bat
disable_heavy_models.bat
```

### Как это работает

- модель и обучение живут на ПК;
- VPS не хранит тяжёлую GPT-модель;
- когда bridge включён, VPS шлёт запросы на ПК;
- когда bridge выключен, бот продолжает жить без тяжёлой GPT-части.

### Как настроить bridge с нуля

Если ты его ещё ни разу не настраивал, не нужно запоминать отдельные команды.

Сделай так:

1. Открой [Центр управления ботом.bat](D:/dis-bot/%D0%A6%D0%B5%D0%BD%D1%82%D1%80%20%D1%83%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F%20%D0%B1%D0%BE%D1%82%D0%BE%D0%BC.bat).
2. Выбери пункт `6. Пошагово: как работает и как настроить bridge`.
3. Потом выбери пункт `9. Включить тяжелые модели с ПК для VPS`.
4. Введи `Tailscale IP` своего ПК.
5. Введи токен bridge.

Если хочешь просто проверить локальный bridge без VPS:

1. Выбери пункт `7. Запустить bridge только локально на ПК`.
2. Когда закончишь, выбери пункт `8. Остановить локальный bridge на ПК`.

## Известные ограничения

- голосовые audio-функции Discord требуют `PyNaCl`;
- тяжёлые GPT-модели не рассчитаны на хранение и обучение на текущем VPS;
- репозиторий не должен использоваться для хранения баз и моделей;
- ежедневный источник правды по сообщениям должен быть один, и сейчас это VPS.
