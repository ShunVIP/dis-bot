## Установка и запуск

### Требования

- Windows (рекомендуется)
- Python **3.12+**
- Discord Bot Token

### Шаги

```bash
# 1. Клонировать репозиторий
git clone https://github.com/ShunVIP/dis-bot.git
cd dis-bot

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Создать папку для БД (если не существует)
mkdir datebase

# 4. Создать файл с токеном
echo tok=YOUR_DISCORD_BOT_TOKEN > KGTD.env

# 5. Запустить
python main_file.py
# или на Windows:
start.bat
```

> ⚠️ Папка для SQLite называется **`datebase/`** — именно так, не `database/`. Создаётся автоматически при первом запуске модулей.

### Обязательные интенты в Discord Developer Portal

- `Message Content Intent`
- `Server Members Intent`

## Функционал

### 💰 Экономика

Монеты — внутренняя валюта сервера. Начисляются за дэйлик и победы в играх. Каждое начисление логируется в `coin_ledger` — полная история транзакций.

| Команда | Описание |
|---------|----------|
| `/баланс [@user]` | Показать баланс монет (свой или чужой) |
| `/дэйлик` | Забрать ежедневную награду (сброс по МСК) |
| `/топ_баланс` | Топ-10 кошельков среди участников сервера |
| `/топ_серии` | Топ-10 по текущей серии дэйлика |

**Формула награды за дэйлик:**
- Базовая: 25 монет
- За каждый день серии (первые 7): +5 монет/день (макс. +35)
- Бонус за серии: +25 монет при достижении 7 / 14 / 30 / 60 / 100 дней подряд

---

### 🏅 Ачивки

Событийная система. Ачивки выдаются автоматически при наступлении условий через `events_bus`. Повторно не выдаются (`UNIQUE(user_id, code)`). Ни один cog не выдаёт ачивку напрямую — только через `emit(...)`.

| Команда | Описание |
|---------|----------|
| `/ачивки [@user]` | Показать список полученных ачивок с датами |

**Список ачивок:**

| Код | Название | Условие |
|-----|----------|---------|
| `first_daily` | Первый дейлик | Первое получение дэйлика |
| `streak_3` | Серия 3 | 3 дня подряд |
| `streak_7` | Серия 7 | 7 дней подряд |
| `streak_14` | Серия 14 | 14 дней подряд |
| `streak_30` | Серия 30 | 30 дней подряд |
| `first_win_rps` | Первая победа в КНБ | Победа в КНБ (бот или PvP) |
| `first_win_guess` | Первая победа в угадайке | Угадать число |
| `first_rep_given` | Первая выданная репутация | Дать `/репа` кому-либо |

**События системы ачивок:**

| Событие | Эмитер | Payload |
|---------|--------|---------|
| `daily_claimed` | `daily.py` | `user_id`, `streak`, `amount` |
| `game_win` | `games.py` | `user_id`, `game` |
| `rep_given` | `rep_and_mood.py` | `user_id`, `given_by`, `date` |

### 🎮 Игры

#### КНБ с ботом

| Команда | Описание |
|---------|----------|
| `/кнб <выбор>` | Камень-ножницы-бумага против бота. Победа = +10 монет |

#### Угадай число

| Команда | Описание |
|---------|----------|
| `/угадай <число> [до]` | Угадать число от 1 до N. Чем больше диапазон — тем выше награда |

Бонус за диапазон: `до >= 50` → +10, `до >= 200` → +20, `до >= 1000` → +40.

#### PvP-дуэли КНБ

Асинхронные дуэли между участниками. Ходы делаются приватно (ephemeral), результат публикуется в канал когда оба сходили. Победитель получает +15 монет.

| Команда | Описание |
|---------|----------|
| `/кнб_дуэль <оппонент> [таймаут_мин]` | Создать дуэль (по умолчанию 15 мин на ход) |
| `/кнб_ход <дуэль_id> <выбор>` | Сделать скрытый ход |
| `/кнб_отмена <дуэль_id>` | Отменить свою незавершённую дуэль (только создатель) |

Статусы дуэли: `open` → `done` / `cancelled` / `expired`.

### 🎂 Дни рождения

Хранение дат, пользовательское и админское управление. Ежедневное автопоздравление в **09:00 МСК** в канал `#flood` (или системный канал). Корректно обрабатывает 29 февраля — в невисокосный год поздравляет одновременно 28 и 29 февраля.

| Команда | Описание |
|---------|----------|
| `/др <ДД.ММ>` | Установить свой день рождения |
| `/д-р` | Удалить свой день рождения |
| `/все_др` | Список всех дней рождения на сервере |
| `/когда_др [@user]` | Дата рождения пользователя (или своя) |
| `/др_ад <@user> <ДД.ММ>` | *(Админ)* Установить ДР другому пользователю |
| `/д-р_ад <@user>` | *(Админ)* Удалить ДР пользователя |

### ⏰ Напоминания и счётчики

Все данные хранятся в `reminders.db` и переживают рестарт бота. APScheduler восстанавливает задачи при старте — просроченные удаляются автоматически.

#### Напоминания

| Команда | Описание |
|---------|----------|
| `/напомни <текст> [через_минут] [дата_время] [лично]` | Создать напоминание. Время в МСК. `лично=True` → в ЛС |
| `/мои_напоминания` | Список активных напоминаний (до 15) |
| `/удалить_напоминание <id>` | Удалить своё напоминание по ID |

#### Счётчики дней до события

| Команда | Описание |
|---------|----------|
| `/счетчик <событие> <ГГГГ-ММ-ДД>` | Разовый расчёт: сколько дней до события |
| `/счетчик_создать <событие> <ГГГГ-ММ-ДД>` | Постоянный счётчик, хранится в БД |
| `/мои_счетчики` | Список своих счётчиков с актуальным отсчётом |
| `/счетчик_удалить <id>` | Удалить счётчик по ID |

### 🎭 Временные роли

Выдача ролей «на время» с автоматическим снятием. Роль создаётся если не существует, поднимается максимально высоко в иерархии для видимости, отображается отдельно в списке участников (`hoist`).

| Команда | Описание |
|---------|----------|
| `/выдать_роль <@user> <название> [минут] [цвет]` | Выдать временную роль (по умолчанию 60 мин). Цвет: HEX или имя (red, blue…) |
| `/активные_роли` | Список активных временных ролей с кнопками управления |
| `/очистить_сироты` | Удалить устаревшие записи из БД |

> Требуются права **Manage Roles**.

### 📊 Статистика сообщений и войса

#### Текстовая статистика

Индексация истории канала с сохранением суточных агрегатов (`messages`, `words`, `emojis`, `chars`). Поддерживает чекпоинты — повторная индексация добавляет только новые сообщения.

| Команда | Описание |
|---------|----------|
| `/индекс_сообщений [канал] [макс_дней]` | Проиндексировать историю канала (по умолчанию 180 дней) |
| `/топ_актив [дней] [канал]` | Топ-15 по количеству сообщений |
| `/топ_слова [дней] [канал]` | Топ-15 по количеству слов |
| `/топ_эмодзи [дней] [канал]` | Топ-15 по использованию эмодзи (Unicode + кастомные) |

#### Голосовая статистика

Онлайн-трекер через `on_voice_state_update` — отслеживает вход, выход и переходы между каналами.

| Команда | Описание |
|---------|----------|
| `/voice_топ [дней]` | Топ-15 по времени в голосе (формат: Xч Yм) |
| `/voice_я [дней]` | Моё суммарное время в войсе за N дней (приватно) |

> Требуется `Message Content Intent` для подсчёта слов и эмодзи.

### 🗃️ WWM Knowledge Base

ETL-система для сбора, хранения и поиска по базе знаний игры **Where Winds Meet**.

**Источники:**
- Fandom Wiki — категории: Character, Locations, Items, Quests (MediaWiki API)
- Game8 Archives — до 800 страниц

**Слои данных:**
1. **RAW-слой** (`raw_records`) — данные «как есть», source of truth для переобработки. Не правится вручную.
2. **Normalized v1** (`entities`, `entity_sources`, `aliases`) — для поиска и slash-команд

**Ежедневное автообновление** в 00:00 по Europe/Berlin.

| Команда | Описание |
|---------|----------|
| `/wwm_search <query>` | Поиск по базе. До 5 результатов: тип сущности, сниппет, ссылки на источники |

**Ручные скрипты:**

| Скрипт | Назначение |
|--------|-----------|
| `run_wwm_refresh_once.py` | Ручной запуск полной сборки |
| `inspect_wwm_db.py` | Проверка RAW-слоя |
| `postprocess_v1.py` | Пересборка normalized из RAW |
| `reset_normalized_layer.py` | Сброс normalized (RAW сохраняется) |
| `inspect_entities.py` | Проверка entities |
| `wwm_search.py` | CLI-поиск по wwm.db |

**Событие после обновления:**

| Событие | Payload |
|---------|---------|
| `wwm_kb_refresh_completed` | `run_id`, `inserted`, `errors`, `postprocess_status` |

## Архитектурные принципы

- **Автозагрузка cogs** — все `.py`-файлы в `fun_slesh/` подключаются при старте без регистрации вручную.
- **Event Bus** — слабая связь между модулями. Cogs не вызывают логику друг друга напрямую — только через `emit/subscribe`. Ошибки обработчиков не останавливают бота.
- **SQLite first** — никакой внешней инфраструктуры (Redis, Postgres и т.д.).
- **MSK-логика времени** — дэйлики, настроение, напоминания, репутация привязаны к московскому времени. Хранение — UTC.
- **Стабильность базы** — существующие таблицы и сигнатуры событий не ломаются. Новый функционал — только новые модули и надстройки.
- **Устойчивость к рестартам** — APScheduler восстанавливает задачи из БД при старте, просроченные удаляются автоматически.
- **Офлайн-first для внешних API** — при недоступности ключей или сервисов бот продолжает работать в офлайн-режиме с информативными сообщениями.

## Дорожная карта

Текущая версия (v1.0.0) является стабильным фундаментом. Все нижеперечисленные направления реализуются как **отдельные модули поверх**, не затрагивая существующий код.

### Запланированные спринты

**Спринт 1 — League of Legends интеграция**
- Линковка аккаунтов через Riot API (`RIOT_API_KEY` в `KGTD.env`)
- Загрузка и хранение истории матчей в `social.db`
- Команды: `/lol_аккаунт`, `/lol_матчи`, `/lol_сводка`
- ML: WinPredictor (LogReg → RandomForest), BuildRecommender
- Команды: `/lol_прогноз_победы`, `/lol_совет_чемпиона`
- Офлайн-режим при отсутствии `RIOT_API_KEY`

**Спринт 2 — Helldivers 2 War Tracker**
- Снапшоты фронта каждые 10–15 мин (публичные community API)
- Команды: `/hd2_фронт`, `/hd2_операции`, `/hd2_прогноз`
- ML: WarForecast — предсказание изменения фронта (Ridge → GBM)

**Спринт 3 — LFG и матч-репорты**
- `/lfg` — поиск группы с кнопками, авто-истечением и интеграцией временных ролей
- `/матч_создать`, `/матч_результат`, `/рейтинг` — с подтверждением соперником и анти-фродом
- `/челлендж_мне`, `/челлендж_сдать`, `/челлендж_рейты`

**Спринт 4 — Боевой пропуск и расширенные ачивки**
- `/паспорт`, `/паспорт_миссии`, `/паспорт_награды`
- Расширенная система ачивок и роли-титулы

**Дальнейшие направления**
- **Контент-Хаб WWM**: автоматическая агрегация гайдов, статов, рецептов; Smart Recognition (OCR/ML) для распознавания скриншотов
- **Авто-отчёты сообщества**: ночные сводки активности, настроения, статистики игр в канал
- **RU-локализация WWM KB**: слой перевода на этапе postprocess + глоссарий «не переводим»
- **Семантический поиск и ML-классификация** сущностей WWM
- **Модерация**: тональность/токс-классификация (офлайн, sklearn), анти-спам по аномалиям активности
