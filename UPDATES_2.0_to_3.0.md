# UPDATES_2.0_to_3.0

Что добавлено **поверх 2.0 Stable** на пути к версии 3.0 (пока реализуем проект “Update на будущее”).

## Главное правило

**2.0 считается стабильной базой.** Мы добавляем новый функционал поверх, не ломая существующую логику 2.0.

## 1) Контур WWM KB (Where Winds Meet Knowledge Base)

Добавлен ingestion-контур для накопления базы знаний по игре **Where Winds Meet**.

### Источники
- **Fandom Wiki (MediaWiki API)** — структурные страницы
- **Game8** — статьи вида `/games/Where-Winds-Meet/archives/<id>`

### Хранилище
- RAW SQLite: **`datebase/wwm.db`**

### Расписание
- Ежедневный запуск в **00:00 (Europe/Berlin)**

### Слои данных
1) **RAW-слой**: таблица `raw_records` — сохраняем данные «как есть» (source of truth для парсинга/переобработки)  
2) **Normalized v1** (postprocess):
   - `entities`
   - `entity_sources`
   - `aliases`
   - `refresh_runs`

### Инструменты/скрипты (ручные)
- `run_wwm_refresh_once.py` — ручной запуск сборки
- `inspect_wwm_db.py` — быстрая проверка RAW
- `postprocess_v1.py` — сборка normalized v1 из RAW
- `reset_normalized_layer.py` — сброс normalized-слоя (RAW не трогает)
- `inspect_entities.py` — проверка normalized-слоя

## 2) Локализация (фундамент)

- **EN** — source of truth (как в источниках).
- **RU** будет добавляться **пакетно на этапе postprocess** (слой перевода рядом с EN).
- Дополнительно будет **глоссарий “не переводим”** — список игровых терминов, которые оставляем в оригинале (EN). Реализуем позже, когда придёт время.

## 3) Структурные договорённости

- Папка для SQLite БД в проекте: **`datebase/`**
- Интеграция новых контуров делается как добавление модулей/задач, не ломая базовые модули 2.0.
