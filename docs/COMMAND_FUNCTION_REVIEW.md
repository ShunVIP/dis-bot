# COMMAND FUNCTION REVIEW

Цель документа: пояснить каждую реализованную Discord slash-команду и подготовить сокращение команд без потери функционала.

Текущее решение по видимости:

- публично в Discord через `/` должны отображаться только `/команды` и `/админ`;
- остальные команды сохраняются во внутреннем каталоге `bot.menu_catalog_commands`;
- перед `bot.tree.sync()` они удаляются из публичного `bot.tree`;
- `/команды` и `/админ` становятся главными точками входа;
- дальнейший этап - заменить выбор скрытых команд на кнопки, select menus и modals там, где нужны параметры.

Рекомендации:

- `Оставить` - команда важна как быстрый Discord-действие.
- `Объединить` - функционал нужен, но команд слишком много; лучше собрать под одну группу/меню.
- `В админку/сайт` - лучше вынести из Discord slash-команд в web/admin panel.
- `Скрыть/админ` - оставить только администраторам.
- `Кандидат на удаление` - можно убрать или оставить как фан-опцию после обсуждения.

| `/админ` | Живой каталог админских команд. | bot tree | Оставить только администраторам. |

Сокращение: меню можно сделать главным входом, а редкие команды скрыть/перенести.

## 19. Главные кандидаты на сокращение

Без потери функционала можно сократить видимый slash-список так:

1. Все настройки (`налог`, `магазин_добавить`, `награды`, `активности`, `итоги канал`, `токсичность`, `болтовня`, `voice_roles`, `релизы`) перенести в web admin.
2. Все maintenance-команды (`собрать_сообщения`, `сбросить_чекпоинты`, `дообучить`, `профилактика`, `индекс_сообщений`, `модели_статус`) оставить только в админке/GUI.
3. Все технические команды игр (`кнб_ход`, `кнб_отмена`, `виселица_буква`) заменить кнопками.
4. Топы объединить в одну группу `/топ` или `/статистика`.
5. Пародию объединить в группу `/пародия`.
6. Репутацию объединить в группу `/репа`.
7. Фан-команды объединить в `/фан` или оставить только через `/команды`.
